Architecture Overview

Fixtures
Dependency injection for all page objects, config, logger, and screenshots. Every test gets a clean, typed context.
Page Objects
6 page classes extend BasePage. Each owns its own locators (data-test selectors) and assertion methods. Zero duplication.
Components
HeaderComponent is shared across all authenticated pages (cart badge, menu, logout). Reused by all POMs and tests.
Constants
Single source of truth for every data-test selector, URL path, page title, and expected text string.
Data Layer
Product catalogue and expected order totals live in src/data. Tests never hardcode prices or product names.
Config
Typed EnvironmentConfig resolved from .env.staging. No process.env reads inside tests or page objects.
Utils
Logger (structured, levelled, auto-attached to report) and ScreenshotUtil (auto-named, auto-attached) injected via fixtures.

Commands

Installation

npm install npx playwright install --with-deps

Run Tests

# All browsers, headless (default) npm test # Single browser npm run test:chrome # Chromium npm run test:firefox # Firefox npm run test:webkit # WebKit/Safari # Headed (visible browser) npm run test:headed # Interactive UI mode npm run test:ui # Debug mode (step-through) npm run test:debug

View Reports

# Open interactive HTML report npm run report # Open Playwright Trace Viewer (on failure) npx playwright show-trace test-results/**/*.zip

Switch User Persona

# Edit .env.staging — no test code changes needed TEST_USERNAME=performance_glitch_user # one of: standard_user, locked_out_user, TEST_USERNAME=problem_user # problem_user, performance_glitch_user, TEST_USERNAME=error_user # error_user, visual_user

Playwright Best Practices Applied

PracticeImplementation
Resilient locatorsAll selectors use data-test attributes — stable across style changes
No arbitrary waitsZero waitForTimeout calls — auto-waiting and explicit waitForURL/waitForSelector
FixturesFull DI via custom test fixture — no manual page object instantiation in tests
Parallel executionfullyParallel: true in config; workers configurable
Multi-browser4 browser projects: Chromium, Firefox, WebKit, Mobile Chrome
No hardcoded dataAll credentials, checkout data, URLs, selectors externalised
Reusable assertionsAssertion methods on each Page Object — not duplicated in tests
CI-readyJUnit XML, trace, video on failure; forbidOnly in CI mode
